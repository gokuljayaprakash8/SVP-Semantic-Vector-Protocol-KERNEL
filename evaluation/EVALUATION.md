# SVP Kernel First-pass Adversarial Evaluation

## Objective

Evaluate the SVP Kernel against a manually curated adversarial dataset representing common AI agent workflow security scenarios.

---

## Dataset

510 manually labeled workflow actions.

Coverage includes:

- Database destruction
- Privilege escalation
- Authentication bypass
- Data exfiltration
- Audit log tampering
- Prompt injection variants
- Agent tool misuse
- MCP-style capability abuse
- Multi-agent workflow risks
- Benign operational workflows

---

---

## Interpretation

This first-pass benchmark prioritizes detecting potentially dangerous workflows.

The current policy configuration achieves high recall, blocking most malicious actions while producing a relatively high false-positive rate. This behavior is expected for an early-stage safety-oriented policy engine and provides a measurable baseline for future policy refinement.

Future iterations will focus on reducing false positives while maintaining strong recall through improved policies, expanded datasets, and iterative benchmarking.

---

## Reproducibility

Evaluation is fully automated through GitHub Actions.

Results are generated by executing:

```
python evaluation/run_eval.py
```

against the published adversarial evaluation corpus.
